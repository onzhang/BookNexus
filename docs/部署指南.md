# BookNexus 部署指南

> 本文档涵盖 BookNexus 图书管理系统的完整部署流程，包含环境准备、中间件部署、应用配置与启动步骤。

---

## 二、中间件部署

### 2.1 MySQL（本地或独立服务器）

```bash
# 安装 MySQL 8.0（以 CentOS 为例）
sudo yum install mysql-server
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 登录并创建数据库
mysql -u root -p
CREATE DATABASE booknexus CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2.2 Redis（本地或独立服务器）

```bash
# 安装 Redis 7.x
sudo yum install redis
sudo systemctl start redis
sudo systemctl enable redis

# 验证连接
redis-cli ping  # 应返回 PONG
```

### 2.3 VM 端中间件一键部署（RabbitMQ + ES + MinIO）

项目根目录提供了 `docker-compose-vm.yml`，适用于在虚拟机或独立服务器上快速部署剩余中间件。

```bash
# 复制配置文件到目标服务器
scp docker-compose-vm.yml root@192.168.100.128:/opt/booknexus/

# 在目标服务器上启动
ssh root@192.168.100.128
cd /opt/booknexus
docker compose -f docker-compose-vm.yml up -d

# 查看运行状态
docker compose -f docker-compose-vm.yml ps
```

**docker-compose-vm.yml 包含的服务：**

| 服务 | 端口 | 说明 |
|------|------|------|
| RabbitMQ | 5672 / 15672 | AMQP 端口 / 管理控制台 |
| Elasticsearch | 9200 / 9300 | REST API / 节点通信 |
| MinIO | 9000 / 9001 | S3 API / 管理控制台 |

> **注意：** 首次启动后请修改 RabbitMQ 和 MinIO 的默认密码。

## 三、应用配置

### 3.1 配置文件说明

| 配置文件 | 用途 |
|----------|------|
| `application.yml` | 主配置（通用配置、Knife4j、XXL-Job） |
| `application-dev.yml` | 开发环境（本地 MySQL/Redis，VM 中间件） |
| `application-test.yml` | 测试环境（独立数据库实例） |
| `application-prod.yml` | 生产环境（环境变量注入，敏感信息不硬编码） |

### 3.2 生产环境配置（application-prod.yml）

生产环境所有敏感信息通过**环境变量**注入，禁止在配置文件中硬编码密码。

**必需的环境变量：**

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `DB_USERNAME` | MySQL 用户名 | `booknexus` |
| `DB_PASSWORD` | MySQL 密码 | — |
| `REDIS_HOST` | Redis 主机地址 | `localhost` |
| `REDIS_PASSWORD` | Redis 密码（如开启） | — |
| `RABBITMQ_HOST` | RabbitMQ 主机地址 | `192.168.100.128` |
| `RABBITMQ_USER` | RabbitMQ 用户名 | `guest` |
| `RABBITMQ_PASSWORD` | RabbitMQ 密码 | — |
| `ES_URIS` | Elasticsearch 地址 | `http://192.168.100.128:9200` |
| `MINIO_ENDPOINT` | MinIO 服务端点 | `http://192.168.100.128:9000` |
| `MINIO_ACCESS_KEY` | MinIO Access Key | `minioadmin` |
| `MINIO_SECRET_KEY` | MinIO Secret Key | — |
| `XXL_JOB_ADMIN` | XXL-Job 调度中心地址 | `http://job-center:8080/xxl-job-admin` |

**启动前设置环境变量：**

```bash
export DB_USERNAME=booknexus
export DB_PASSWORD=your_secure_password
export REDIS_HOST=localhost
export REDIS_PASSWORD=your_redis_password
# ... 其他变量
```

或使用 `.env` 文件配合 `source .env` 加载。

### 3.3 数据库初始化

```bash
# 执行 DDL + 索引 + 种子数据
mysql -u root -p booknexus < sql/init.sql
```

> 默认管理员账号：`admin` / `a123456`

## 四、构建与启动

### 4.1 编译打包

```bash
# 开发环境编译
mvn clean compile

# 运行单元测试
mvn clean test -Pdev

# 生产环境打包
mvn clean package -Pprod -DskipTests
```

打包成功后，产物位于 `target/booknexus-1.0.0.jar`。

### 4.2 直接启动（开发环境）

```bash
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

或启动已打包的 JAR：

```bash
java -jar -Dspring.profiles.active=dev target/booknexus-1.0.0.jar
```

### 4.3 生产环境启动

```bash
# 设置环境变量后启动
export SPRING_PROFILES_ACTIVE=prod
export DB_USERNAME=booknexus
export DB_PASSWORD=your_secure_password
# ... 设置其余环境变量

nohup java -Xms512m -Xmx1024m -jar booknexus-1.0.0.jar > app.log 2>&1 &
```

**JVM 参数建议：**

| 场景 | 推荐参数 |
|------|---------|
| 2C4G 服务器 | `-Xms512m -Xmx1024m` |
| 4C8G 服务器 | `-Xms1024m -Xmx2048m` |
| 高并发场景 | 增加 `-XX:+UseG1GC -XX:MaxGCPauseMillis=200` |

## 六、前端部署

前端为独立的 Vue 3 项目，位于 `frontend/` 目录。

### 6.1 构建

```bash
cd frontend
npm install
npm run build
```

构建产物位于 `frontend/dist/`。

### 6.2 部署到 Nginx

```nginx
server {
    listen 90;
    server_name localhost;
    root /usr/share/nginx/html/booknexus;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 七、常见问题

### 7.1 启动失败：数据库连接超时

**现象：** `Communications link failure`

**解决：**
1. 检查 MySQL 是否启动：`systemctl status mysqld`
2. 检查网络连通性：`telnet mysql-host 3306`
3. 检查 `allowPublicKeyRetrieval=true` 是否配置（开发环境）

### 7.2 Redis 连接失败

**现象：** `Connection refused`

**解决：**
1. 检查 Redis 是否启动：`systemctl status redis`
2. 检查 `bind` 配置是否允许远程连接
3. 检查防火墙是否放行 6379 端口

### 7.3 RabbitMQ / ES / MinIO 连接失败

**现象：** 对应中间件启动报错

**解决：**
1. 检查 VM 端 Docker 服务是否正常运行：`docker ps`
2. 检查网络是否可达：`ping 192.168.100.128`
3. 检查端口是否放行：`telnet 192.168.100.128 5672`

### 7.4 Sentinel 限流触发

**现象：** 接口返回 `429 请求过于频繁，请稍后再试`

**解决：**
1. 检查对应资源的 QPS 阈值是否过低（见 `SentinelRuleInitializer`）
2. 高并发场景下可适当调高阈值或水平扩容
3. 生产环境建议接入 Sentinel 控制台动态调整规则

### 7.5 缓存未生效

**现象：** `getById` 查询未走 Redis 缓存

**解决：**
1. 确认 `@EnableCaching` 已开启（`RedisConfig` 已配置）
2. 检查 Redis 是否正常运行：`redis-cli ping`
3. 检查 `spring.cache.type=redis` 是否配置

### 7.6 XXL-Job 执行器注册失败

**现象：** 启动日志出现 `xxl-job admin-addresses empty`

**解决：**
- 此为正常行为。本地开发时 `admin-addresses` 为空会自动跳过执行器注册，不影响业务功能。
- 生产环境需配置 `XXL_JOB_ADMIN` 环境变量指向调度中心地址。

## 八、升级与维护

### 8.1 平滑升级

```bash
# 1. 拉取最新代码
git pull origin master

# 2. 重新打包
mvn clean package -Pprod -DskipTests

# 3. 备份旧版本
cp booknexus-1.0.0.jar booknexus-1.0.0.jar.bak

# 4. 替换新版本并重启
mv target/booknexus-1.0.0.jar .
bash restart.sh
```

### 8.2 日志查看

```bash
# 实时查看应用日志
tail -f logs/booknexus.log

# 查看错误日志
grep ERROR logs/booknexus.log
```

日志配置采用 30 天轮转策略，详见 `logback-spring.xml`。
